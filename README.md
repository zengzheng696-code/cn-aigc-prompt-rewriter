**仓库布局与链接：** 若 **`README.md` 在仓库根目录**，Skill 文件在子目录 **`cn-aigc-prompt-rewriter/`** 内（与常见 GitHub 结构一致），下文链接均写成 **`cn-aigc-prompt-rewriter/文件名`**。  
若你 **把整个 Skill 文件夹单独当作仓库根**（`README.md` 与 `SKILL.md` 同级），请自行把链接里的 **`cn-aigc-prompt-rewriter/`** 前缀去掉。

## 价值：为什么要用这个 Skill

用语言去描述 **图像或视频**（两种完全不同的模态），中间往往会伴随 **九成以上的信息缺失**：你心里想的构图、光线、材质、层次和节奏，很难在一两句话里说全，模型只能「猜」。这正是 **文生图 / 文生视频** 特别容易陷入 **反复抽卡、大量作废出图** 的根本原因——不是运气差，而是 **输入里该有的视觉指令根本没写够、没写对**。

本 Skill 要做的，是把你的 **原始提示词当成半成品**：在此基础上 **拓展、润色与结构化**，并按 **GPT Image、Nano Banana、Seedance、Veo** 等不同模型的写法偏好做 **适配**，让模型更少瞎猜、更少跑偏，从而 **压低无效生成次数，把时间 Token 花在更接近成片的方向上**。

这是一套面向各类 **AI 助手 / 编码 Agent** 的可执行规则：在真正调用生图、生视频模型之前，先把中文草稿整理成 **可直接粘贴进模型**、且 **带模型标签与多版本对比** 的成稿。

> **`SKILL.md` 顶部的 YAML**（`name` / `description`）主要为 **Cursor Skills** 的发现机制设计；在其他工具里可 **忽略 frontmatter**，正文规则仍然全部适用。

## 能力概览

| 能力 | 说明 |
|------|------|
| **首轮输出** | 默认 **恰好 3 条**（版本 A/B/C），在构图、光线或镜头节奏上有 **肉眼可见差异**，核心意图保持一致 |
| **模型标注** | 每条标题包含 **「适用于：具体模型」**，避免贴错工具链 |
| **覆盖模型** | 图像：**GPT Image 2.0**、**Nano Banana（Gemini 图）**；视频：**Seedance**、**Veo** |
| **修订** | 指定某一版继续改 → 单条 **「版本 X · 修订 · 适用于：<模型>」** 全文 |
| **跨模型适配** | 例如 Nano ↔ GPT Image、Seedance ↔ Veo，按目标模型 **重写结构**，保留创意内核 |
| **模板与依据** | [`references/`](cn-aigc-prompt-rewriter/references/) 内提供图/视频骨架及与 **厂商文档对齐** 的说明链接 |

执行细则与质量门见 **[SKILL.md](cn-aigc-prompt-rewriter/SKILL.md)**。下文 **示例一至四** 已附逐字成稿；雨夜街景、小红书封面、Veo 口播、素材绑定等扩展稿在同目录文件 `cn-aigc-prompt-rewriter/examples.md` 中。

## 输出格式约定（首轮 / 修订 / 适配）

**首轮：** 必须输出三条；标题为 `【版本 A · 均衡 · 适用于：<模型>】` 等形式（模型为 GPT Image 2.0 / Nano Banana / Seedance / Veo 之一，且与图/视频任务一致）。

**修订：** 用户指定某一版修改时，只输出一条 `【版本 X · 修订 · 适用于：<模型>】` + 全文。

**跨模型适配：** 用户要求换成另一模型时，输出 `【版本 X · 适配 · 适用于：<目标模型>】` + 全文（条数按用户要求）。

**首轮标题示意：**

```text
【版本 A · 均衡 · 适用于：Nano Banana】

<完整提示词正文>
```

（版本 B、C 同理；图任务不出现 Seedance/Veo，视频任务不出现 GPT Image/Nano Banana，除非 SKILL 允许的分版本绑定。）

## 示例说明：从「一句话」到「可执行的视觉指令」

### 典型输入长什么样？

常见原始需求只有 **题材 + 气质词**，缺少 **景别、光线、材质、约束**，例如：

- 「一只猫坐在窗边看窗外」
- 「护肤精华液电商主图，干净高级」
- 「5 秒短视频：女生地铁站看见前任，情绪复杂」

这类句子 **叙事清楚、画面不清楚**：模型不知道你是要特写还是远景、清晨还是雨夜、纪实还是插画，只能靠抽样碰运气。

### 典型输出长什么样？

Skill 会在 **不篡改你本意** 的前提下，补足 **主体 / 场景 / 动作 / 构图 / 光线 / 风格 / 约束**（视频再加 **时长、镜头节拍、声音**），并按所选模型习惯排版（例如 GPT Image 强调 **用途与画中字位置**，Seedance 把 **镜头运动** 和 **演员动作** 拆句写）。

**首轮**固定给出 **A / B / C 三条**，标题里写明 **适用于哪种模型**，三条之间在 **构图或光线或节奏** 上拉开差距，而不是换几个形容词糊弄。

### 示例一：文生图（生活场景）

| | 内容 |
|---|------|
| **你输入** | 一只猫坐在窗边看窗外。 |
| **改写方向（摘要）** | 版本 A 可走 **中景、午后斜阳、安静治愈**；版本 B 换 **俯视广角、阴天冷调、电影静帧感**；版本 C 走 **特写短景深、黄昏金色侧光、突出毛发纹理**。三条都会补上 **画幅、排除水印畸形肢** 等硬约束。 |
| **模型标注** | 未指定时图像任务常标 **Nano Banana**；若指定 GPT Image，则顺序靠拢「用途 → 场景 → 主体 → 细节 → 约束」。 |

**逐字成稿（首轮三条 + 修订一条）：**

**用户输入：**  
一只猫坐在窗边看窗外。

```
【版本 A · 均衡 · 适用于：Nano Banana】

一幅温馨写实图像：一只毛色柔和的猫侧身坐在木质窗台上，望向窗外街道，午后斜阳从左侧落入室内，在毛发边缘形成柔和轮廓光。构图为中景，窗台与窗框形成稳定画框，窗外街道略微虚化。氛围安静治愈，细节包含木纹、纱帘轻摆、空气中微尘。画幅 4:3。避免出现畸形五官、多余肢体、水印与画面文字。

【版本 B · 创意 · 适用于：Nano Banana】

同一主体与动作：猫蹲坐在窗边望向窗外，改用 **俯视略带广角** 构图，强调窗台平面与窗外纵深；光线改为 **阴天漫反射**，整体偏冷绿与灰蓝，情绪更内敛疏离。窗外保留雨天或湿润街面反光其一以增加层次。风格仍是写实但略带电影静帧感。禁止卡通化、禁止主体换成其他动物、禁止添加夸张滤镜文字。

【版本 C · 稳定 · 适用于：Nano Banana】

写实特写：**猫眼部与肩颈为主**，窗外仅占画面一侧条带；固定 **平视短景深**，背景高度虚化仅保留光斑。黄昏金色侧光，毛发纹理清晰，窗台木纹为辅景。明确排除水印、文字、畸形爪子、画面污渍。
```

**用户跟进：**  
把版本 B 改成黄昏暖光，窗外要是晴天。

```
【版本 B · 修订 · 适用于：Nano Banana】

同一主体与动作：猫蹲坐在窗边望向窗外，维持俯视略带广角构图与室内外纵深。光线改为 **晴朗黄昏暖金色侧光**，室内墙面反射暖调，窗外为 **晴天街景**（可见清晰天际线与柔和云彩），路面可有干燥高光而非雨天反光。情绪由阴冷改为温暖怀旧，仍为写实电影静帧感。禁止卡通化、禁止更换主体、禁止水印与画面文字。
```

### 示例二：文生图（商业 / 文字）

| | 内容 |
|---|------|
| **你输入** | 护肤精华液电商主图，干净高级；或海报要带固定文案「春日咖啡节」。 |
| **改写方向（摘要）** | 商业图会写清 **背景（如无缝白）、棚光、材质（玻璃/泵头）、比例**；带字海报会写 **引号内精确字面、字体气质、版面三区、禁止乱码替换字**。 |
| **模型倾向** | **密集排版 / 固定字迹** 时常优先 **GPT Image 2.0** 写法（见 [`image-templates.md`](cn-aigc-prompt-rewriter/references/image-templates.md)）。 |

**逐字成稿（电商主图）：**

**用户输入：**  
给我做一张护肤精华液的电商主图，干净高级一点。

```
请生成一张 4:5 的高端写实产品图。主体为一瓶透明玻璃质感的护肤精华液，放置在纯白无缝背景中央，瓶身标签清晰可读，产品边缘高光干净。构图采用中近景正视角，前景仅保留轻微柔焦反射，不添加多余道具。光线使用柔和棚拍主光与弱轮廓光，整体色调为冷白与浅银，强调玻璃、液体与金属泵头的材质细节。画面风格简洁、商业、电商平台可直接使用。避免出现人物、手部、复杂背景、夸张光斑、水印、错字标签。
```

**逐字成稿（海报含固定文案）：**

**用户输入：**  
做一张咖啡节海报，要有文字“春日咖啡节”。

```
请生成一张 3:4 竖版活动海报，主题为春季咖啡节。主体画面为木质桌面上的手冲咖啡与花束，背景是柔和晨光与浅绿色植物虚化。构图采用上中下三段式：上方留主标题区，中部为主体静物，下方留活动信息区。主标题必须出现文字“春日咖啡节”，字体气质温暖文艺、字形清晰、边缘干净，放置在画面上方居中。整体配色为奶油白、浅棕、鼠尾草绿，风格清新、治愈、可用于线下活动宣传。避免出现英文替换字、乱码、拥挤排版、水印、无关人物。
```

### 示例三：文生图（编辑）

| | 内容 |
|---|------|
| **你输入** | 基于这张客厅图，白天改傍晚，其它别动。 |
| **改写方向（摘要）** | 明确 **Preserve（布局、家具、视角不变）** 与 **Change（仅光线时段）**，避免模型自由发挥改构图。 |

**逐字成稿：**

**用户输入：**  
基于我这张客厅图，把白天改成傍晚氛围，其他别动。

```
保持原图客厅的空间布局、家具位置、镜头视角与主体结构不变，只将时间氛围从白天调整为傍晚。将窗外光线改为偏暖金橙色，室内阴影略加深，墙面与木地板反射呈现柔和暖调。保留原有材质细节与清晰度，不新增或删除任何家具与装饰，不改变构图比例，不引入风格漂移。
```

### 示例四：文生视频（分镜与时间轴）

| | 内容 |
|---|------|
| **你输入** | 地铁站遇见前任，情绪复杂，大概 5 秒。 |
| **改写方向（摘要）** | 拆 **场景 → 主体表演 → 镜头（各几秒）→ 环境声**，必要时 **对白引号**；Seedance 写法会把 **运镜句** 与 **表演句** 分开，减少指令打架。 |

**逐字成稿：**

**用户输入：**  
写一个5秒短视频：女生在地铁站看到前任，情绪复杂。

```
请生成一段 5 秒现实主义短剧情视频。场景是晚高峰地铁站台，主体为一位通勤女生在人群中突然停顿，目光锁定远处熟悉身影。镜头第 1-2 秒为中景跟拍并轻微推进，第 3-4 秒切到女生近景表情，眼神从平静转为克制波动，第 5 秒快速切到对面人影背影后停在女生呼吸起伏的特写。冷白站台灯与列车进站风压形成环境压力，背景包含列车提示音与人群脚步声。整体节奏紧凑克制，避免夸张表演、过度慢动作与无关转场特效。
```

### 修订（单条替换）

你对助手说：**「把版本 B 改成黄昏暖光，窗外晴天。」**  
助手只回 **一条** `【版本 B · 修订 · 适用于：…】`，在保留 B 的构图叙事前提下 **只改光线与窗外天气**，不重头推翻三条。**示例一** 末尾已附对应修订逐字稿。

### 跨模型适配

你对助手说：**「把版本 A 改成 GPT Image 2.0 能用的。」**  
助手输出 **一条** `【版本 A · 适配 · 适用于：GPT Image 2.0】`，把同一创意 **按 GPT Image 的顺序与字段重写**（用途、场景背景、主体细节、约束等），而不是简单同义词替换。规则见 **[SKILL.md](cn-aigc-prompt-rewriter/SKILL.md)**。
