# cn-aigc-prompt-rewriter

## 价值：为什么要用这个 Skill

用语言去描述 **图像或视频**（两种完全不同的模态），中间往往会伴随 **九成以上的信息缺失**：你心里想的构图、光线、材质、层次和节奏，很难在一两句话里说全，模型只能「猜」。这正是 **文生图 / 文生视频** 特别容易陷入 **反复抽卡、大量作废出图** 的根本原因——不是运气差，而是 **输入里该有的视觉指令根本没写够、没写对**。

本 Skill 要做的，是把你的 **原始提示词当成半成品**：在此基础上 **拓展、润色与结构化**，并按 **GPT Image、Nano Banana、Seedance、Veo** 等不同模型的写法偏好做 **适配**，让模型更少瞎猜、更少跑偏，从而 **压低无效生成次数，把时间 Token 花在更接近成片的方向上**。

这是一套面向各类 **AI 助手 / 编码 Agent** 的可执行规则：在真正调用生图、生视频模型之前，先把中文草稿整理成 **可直接粘贴进模型**、且 **带模型标签与多版本对比** 的成稿。

> **`SKILL.md` 顶部的 YAML**（`name` / `description`）主要为 **Cursor Skills** 的发现机制设计；在其他工具里可 **忽略 frontmatter**，正文规则仍然全部适用。

## 能力概览

| 能力 | 说明 |
|------|------|
| **首轮输出** | 默认 **恰好 3 条**（版本 A/B/C），在构图、光线或镜头节奏上有 **肉眼可见差异**，核心意图保持一致 |
| **模型标注** | 每条标题包含 **「适用于：具体模型」**，避免贴错工具链 |
| **覆盖模型** | 图像：**GPT Image 2.0**、**Nano Banana（Gemini 图）**；视频：**Seedance**、**Veo** |
| **修订** | 指定某一版继续改 → 单条 **「版本 X · 修订 · 适用于：<模型>」** 全文 |
| **跨模型适配** | 例如 Nano ↔ GPT Image、Seedance ↔ Veo，按目标模型 **重写结构**，保留创意内核 |
| **模板与依据** | `references/` 内提供图/视频骨架及与 **厂商文档对齐** 的说明链接 |

完整规则见 **[`SKILL.md`](SKILL.md)**；更多可复制长示例见 **[`examples.md`](examples.md)**。

## 示例说明：从「一句话」到「可执行的视觉指令」

下面用 **描述性示例** 说明 Skill 在做什么；**逐字成稿**见 [`examples.md`](examples.md)。

### 典型输入长什么样？

常见原始需求只有 **题材 + 气质词**，缺少 **景别、光线、材质、约束**，例如：

- 「一只猫坐在窗边看窗外」
- 「护肤精华液电商主图，干净高级」
- 「5 秒短视频：女生地铁站看见前任，情绪复杂」

这类句子 **叙事清楚、画面不清楚**：模型不知道你是要特写还是远景、清晨还是雨夜、纪实还是插画，只能靠抽样碰运气。

### 典型输出长什么样？

Skill 会在 **不篡改你本意** 的前提下，补足 **主体 / 场景 / 动作 / 构图 / 光线 / 风格 / 约束**（视频再加 **时长、镜头节拍、声音**），并按所选模型习惯排版（例如 GPT Image 强调 **用途与画中字位置**，Seedance 把 **镜头运动** 和 **演员动作** 拆句写）。

**首轮**固定给出 **A / B / C 三条**，标题里写明 **适用于哪种模型**，三条之间在 **构图或光线或节奏** 上拉开差距，而不是换几个形容词糊弄。

### 示例一：文生图（生活场景）

| | 内容 |
|---|------|
| **你输入** | 一只猫坐在窗边看窗外。 |
| **改写方向（摘要）** | 版本 A 可走 **中景、午后斜阳、安静治愈**；版本 B 换 **俯视广角、阴天冷调、电影静帧感**；版本 C 走 **特写短景深、黄昏金色侧光、突出毛发纹理**。三条都会补上 **画幅、排除水印畸形肢** 等硬约束。 |
| **模型标注** | 未指定时图像任务常标 **Nano Banana**；若指定 GPT Image，则顺序靠拢「用途 → 场景 → 主体 → 细节 → 约束」。 |

完整三条正文示例见 [`examples.md`](examples.md) 第 0 节。

### 示例二：文生图（商业 / 文字）

| | 内容 |
|---|------|
| **你输入** | 护肤精华液电商主图，干净高级；或海报要带固定文案「春日咖啡节」。 |
| **改写方向（摘要）** | 商业图会写清 **背景（如无缝白）、棚光、材质（玻璃/泵头）、比例**；带字海报会写 **引号内精确字面、字体气质、版面三区、禁止乱码替换字**。 |
| **模型倾向** | **密集排版 / 固定字迹** 时常优先 **GPT Image 2.0** 写法（见 `references/image-templates.md`）。 |

对应长示例见 [`examples.md`](examples.md) 第 1、2、5 节。

### 示例三：文生图（编辑）

| | 内容 |
|---|------|
| **你输入** | 基于这张客厅图，白天改傍晚，其它别动。 |
| **改写方向（摘要）** | 明确 **Preserve（布局、家具、视角不变）** 与 **Change（仅光线时段）**，避免模型自由发挥改构图。 |

见 [`examples.md`](examples.md) 第 4 节。

### 示例四：文生视频（分镜与时间轴）

| | 内容 |
|---|------|
| **你输入** | 地铁站遇见前任，情绪复杂，大概 5 秒。 |
| **改写方向（摘要）** | 拆 **场景 → 主体表演 → 镜头（各几秒）→ 环境声**，必要时 **对白引号**；Seedance 写法会把 **运镜句** 与 **表演句** 分开，减少指令打架。 |

见 [`examples.md`](examples.md) 第 6、7、8、9 节。

### 修订示例（单条替换）

你对助手说：**「把版本 B 改成黄昏暖光，窗外晴天。」**  
助手只回 **一条** `【版本 B · 修订 · 适用于：…】`，在保留 B 的构图叙事前提下 **只改光线与窗外天气**，不重头推翻三条。

叙述级演示见 [`examples.md`](examples.md) 第 0 节修订段。

### 跨模型适配示例

你对助手说：**「把版本 A 改成 GPT Image 2.0 能用的。」**  
助手输出 **一条** `【版本 A · 适配 · 适用于：GPT Image 2.0】`，把同一创意 **按 GPT Image 的顺序与字段重写**（用途、场景背景、主体细节、约束等），而不是简单同义词替换。

## 首轮标题格式（示意）

```text
【版本 A · 均衡 · 适用于：Nano Banana】

<完整提示词正文>

【版本 B · 创意 · 适用于：Nano Banana】

<完整提示词正文>

【版本 C · 稳定 · 适用于：Nano Banana】

<完整提示词正文>
```

图任务不出现 Seedance/Veo；视频任务不出现 GPT Image/Nano Banana（除非你在 SKILL 允许的前提下做分版本绑定）。
