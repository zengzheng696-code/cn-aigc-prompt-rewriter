# 文生图模板（中文输出）

首轮输出 **三条** 完整提示词时：每条标题须含 **适用于：GPT Image 2.0** 或 **适用于：Nano Banana**（见 `SKILL.md`）。在共用同一用户意图的前提下，让 **A/B/C 分别侧重不同维度**（例如 A 中景自然光、B 特写戏剧性光线、C 远景氛围），避免三条只剩形容词微调。修订与跨模型适配同理在标题中标明适用模型。

本文件用于图像任务改写，支持：
- GPT Image 2.0
- Nano Banana

## A. GPT Image 2.0 模板

适用：文字排版明确、品牌视觉、UI mockup、结构控制强的场景。

与 OpenAI Cookbook 对齐的输出顺序（复杂时可分段标题或换行，不必挤成一句）：

1) **用途 / 成片类型**（电商主图、APP 界面示意、海报印刷等——帮助模型对齐精度预期）  
2) **场景 / 背景**（环境先到，再落到主体）  
3) **主体与关键元素**（动作或状态）  
4) **构图与镜头视角**（景别、机位、前中后景层次）  
5) **光线与色彩、材质与质感**  
6) **文字要求**（如有：引号包住 **精确字面**，并写字体气质、大致位置、可读性；密集小字时在调用侧配合更高 quality，提示词侧写明「小字须清晰」）  
7) **约束**（不出现什么；写实需求可显式写 photorealistic / 写实摄影等，与官方建议一致）  
8) 若为编辑任务，增加 **Change / Preserve / Constraints**（编辑类务必「只改列举项」，Preserve 可重复关键句以防 drift）

参考骨架：

```txt
用途：[电商主图 / UI mock / 海报 …]。请生成一张[画幅/比例]的图像。场景与背景：[环境描述]。主体为[主体]，状态或动作为[…]。构图采用[镜头/机位/景别]，前景[…]、中景[…]、背景[…]。光线为[…]，色调为[…]，材质重点表现[…]，整体质感[写实/插画等]。画面中必须出现文字“[精确文案]”（字体气质[…]，位置[…]，边缘清晰可读）。避免出现[不需要元素列表]。
```

编辑任务附加骨架：

```txt
Change: [只写要改动的1-3项]
Preserve: [主体身份/构图/光线/背景中不允许变化的部分]
Constraints: [不新增无关元素，不改变风格，不改变比例等]
```

## B. Nano Banana 模板

适用：自然场景、写实氛围、快速高性价比出图、极端比例场景；与 Google DeepMind / Cloud Blog 的「具体、正向、强动词」导向一致。

输出结构（贴近官方 Subject + Action + Context + Composition + Style）：

1) **主体**  
2) **动作或情境**（可用强动词起句，如「生成一幅……」）  
3) **场景 / 地点语境**  
4) **构图**（景别、视角、留白）  
5) **光线与天气 / 环境细节**（2～4 个可见细节）  
6) **风格与画幅**；需要写实时可写摄影感、材质真实感等  
7) **画中字**（如有：引号 + 字体气质，与官方一致）  
8) **约束**：主段落以 **正向描写「要什么」** 为主；「不要水印」等硬约束放在句末，避免空洞否定堆砌  
9) 若为编辑任务：单独说明 **只改哪些 / 哪些必须保持不变**

摄影与镜头语言（浅景深、广角感、低角度等）可按官方示例使用；**避免** 在同一提示词里写多套互相打架的参数（例如矛盾的焦段与取景描述）。

参考骨架：

```txt
生成一幅[风格]图像：[主体]在[场景]中[动作/情境]。构图采用[景别与视角]，画面氛围[情绪]，光线为[光线描述]，可见细节包含[2-4项]。画幅比例[比例]，整体质感[关键词]。如需画面文字，必须出现“[文案]”（字体气质[…]）。请以正向语句描述理想画面；末尾约束：避免出现[水印/畸形/多余肢体等具体项]。
```

编辑任务补充句：

```txt
保持[人物外观/主体形状/主构图/色调基线]不变，只调整[指定变化项]。
```

## 文生图改写补全清单

遇到用户输入过短时，至少补全以下字段：
- 主体（谁/什么）
- 场景（在哪里）
- 动作（在做什么）
- 风格（写实/插画/电影感等）
- 光线（时间、方向、强弱）
- 构图（近景/中景/远景，视角）
- 约束（不出现什么）

若是编辑类请求，再补：
- 改动项（Change）
- 保留项（Preserve）
- 强约束（Constraints）
